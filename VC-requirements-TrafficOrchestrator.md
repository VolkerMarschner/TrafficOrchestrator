Project Overview\n\n**Project Name:** Traffic Orchestrator short: TO\n**Language:** preferred Go, but not fixed yet\n**Target Platform:** Linux cli/service, windows cli/Service\n\n### What does it do?\nThis shall be a simple system to create real network flows in a small to medium (up to 100 hosts) demo environment. Flows can be configured with a simple flat config file.\n#### Architecture and Logic\nThe system consists of 2 logical components (Master and Agents), that is ideally contained in one single binary. The binary shall run on the target plattforms without dependencies or installation process.\nThe working principle is as follows: The Master (designated by a config option) and the Agents communicate over a separate channel (secured via a pre shared key) over a single\nTCP or UDP connection with each other.\nOver this channel, the master instructs 2 or more Agents to communicate directly with each other to create traffic based of a defined config that the master has.\ne.g. Master tells \n- Host A: communicate as a client via UDP Port 53 with Host B (wheras communication in this context means only establish a full connection like  3 way handshake and after that tear down the connection) with x number of connections\n- Host B: await a connection (aka open a listening port) on port 53, and after teardown of the connection, wait for x seconds and close the port\n\n#### Example Config file\n\n=============================================================================\n\ntraffic_list.conf – Packet list for sender.sh\n\n=============================================================================\n\nFormat:\n\nPROTOCOL    DESTINATION    PORT    INTERVAL(s)    COUNT\n\n\nPROTOCOL:     TCP | UDP\n\nDESTINATION:  TARGET1..5 or direct IP\n\nPORT:         Port number\n\nINTERVAL:     Seconds between connections\n\nCOUNT:        Number or \"loop\" (default: loop)\n\n=============================================================================\n\nTarget1=192.168.1.100\nTarget2=192.168.1.102\netc.\n--- Fileserver (TARGET1) ---\n\nTCP     TARGET1     445     5       loop    # SMB\n\nTCP     TARGET1     139     10      loop    # NetBIOS\n\n--- Webserver (TARGET2) ---\n\nTCP     TARGET2     80      3       loop    # HTTP\n\nTCP     TARGET2     443     4       loop    # HTTPS\n\n--- DNS (TARGET3) ---\n\nUDP     TARGET3     53      2       loop    # DNS\n\n--- Gateway (TARGET4) ---\n\nTCP     TARGET4     8080    15      loop    # HTTP alt\n\n--- Mail (TARGET5) ---\n\nTCP     TARGET5     25      60      loop    # SMTP\n\nTCP     TARGET5     143     60      loop    # IMAP\n\n\n### What is explicitly out of scope?\n\n- Traffic elements that are content/payload and or Layer5-7 Information\n- \n\n---\n\n## 2.
